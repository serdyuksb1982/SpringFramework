Spring Framework, по шагам, согласно курса Udemy.

	Продолжение изучения Spring:
   Spring Boot. В этой реализации Проекта подразумевается создание множества Entity (сущностей). Для каждгого Entity
нужно будет создавать свой Repository. Чтобы избежать этого, используем Spring Data JPA - механизм для взаимодействия
таблиц БД, позволяющий минимизировать кол-во кода. Для этого extends JpaRepository<Employee, Integer> и используем 
дефолтные методы данного интерфейса (JpaRepository). Методы интерфейса:
findAll, получение всех работников
     * findById, Получение одного работников
     * save, добавление работника, в Reguest добавляется id
     * save, изменение работника
     * deleteById удаление работника
Так же, можно создавать свои методы, соблюдая правильное именование методов, согласно JpaRepository, что и было выполнено
мною в проекте. Те данный способ подходит для небольших и не сложных приложений.
